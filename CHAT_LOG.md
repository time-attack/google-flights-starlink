Google Flights enforces a strict **Trusted Types** security policy. This policy blocks standard scripts from assigning raw HTML strings to `element.innerHTML` or `element.outerHTML` to prevent Cross-Site Scripting (XSS). 

When our original extension script tried to inject the badge SVG or the tooltip HTML, Google Flights intercepted it, threw a security exception, and terminated the script execution. Consequently, no badges were shown at all.

I refactored [content.js](file:///Users/sinmat/Desktop/starlinkonly/content.js) to build all badge SVGs, tooltips, and segment rows **programmatically using standard DOM APIs** (e.g., `document.createElementNS`, `document.createElement`, and `element.appendChild`).
